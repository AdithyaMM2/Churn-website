Customer Churn Prediction using Amazon SageMaker

This project implements an end-to-end machine learning workflow on AWS for telecom customer churn prediction. The solution covers preprocessing, baseline modelling, managed XGBoost training in SageMaker, hyperparameter tuning, S3 artifact storage, and final evaluation on a held-out test set.

The final outcome was intentionally reported honestly: the cloud-trained XGBoost pipeline worked, but Logistic Regression remained the strongest overall model on the test set.

AWS Architecture

The workflow uses an Amazon SageMaker AI JupyterLab Space for experimentation, Amazon S3 for dataset and artifact storage, SageMaker managed training jobs for XGBoost, hyperparameter tuning jobs for optimization, and CloudWatch for job logs. A Batch Transform attempt was made but failed because of an account-level service quota limit.

1

Prepare and preprocess data in SageMaker notebook

Load the churn dataset, clean TotalCharges, remove identifier columns, encode categorical variables, and split data.

2

Upload CSV files to Amazon S3

Store train, validation, and test artifacts in S3 for managed training workflows.

3

Train and tune XGBoost in SageMaker

Launch managed training and hyperparameter tuning jobs using SageMaker’s built-in XGBoost algorithm.

4

Evaluate model artifacts

Download trained artifacts from S3, score the held-out test set in the notebook, and compare against the baseline model.

Final Results

The website reports the final results exactly as observed. The cloud-based pipeline succeeded technically, but the strongest model overall was still the simpler Logistic Regression baseline.

What was built

Why AWS was used

To demonstrate a managed AI/ML ecosystem with cloud storage, managed training jobs, tuning jobs, logging, and reusable model artifacts.
